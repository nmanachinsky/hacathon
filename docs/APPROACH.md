# Подход к решению

## Архитектура конвейера

```
┌──────────┐   ┌─────────┐   ┌──────────┐   ┌──────────┐   ┌────────┐   ┌────────┐
│ Walker   │ → │ Dedup   │ → │ Router   │ → │ Extractor│ → │Detect  │ → │Classify│
└──────────┘   └─────────┘   └──────────┘   └──────────┘   └────────┘   └────────┘
                                                  ↑                          │
                                                  │                          ▼
                                            (process pool)              ┌──────────┐
                                                                        │ Reports  │
                                                                        │ csv/json │
                                                                        │ md/html  │
                                                                        └──────────┘
```

Каждый файл проходит шесть этапов независимо в подпроцессе, что позволяет масштабироваться по ядрам CPU.

## Детекция

Гибридный подход: **regex + алгоритмические валидаторы + контекстный скоринг + опциональный NER (Natasha)**.

| Категория | Алгоритм | Валидатор |
|-----------|----------|-----------|
| ФИО | Regex (Имя Отчество Фамилия + инициалы) + NER (PER) | — |
| Телефон | Regex (RU форматы +7/8) | — |
| Email | Regex (RFC-совместимый) | — |
| Дата рождения | Regex с контекстом «г.р.» / «рождения» | — |
| Адрес | Regex (улицы/города) + NER (LOC) + почтовый индекс с контекстом | — |
| Паспорт РФ | Regex 4+6 цифр + контекст «паспорт/серия» | — |
| СНИЛС | Regex 11 цифр + контрольная сумма | snils_check |
| ИНН | Regex 10/12 цифр + контрольная сумма | inn_check |
| ОГРН | Regex 13/15 + контрольная сумма + контекст | ogrn_check |
| ВУ | Regex 4+6 / 2 буквы + 6 цифр + контекст | — |
| MRZ | Структурное совпадение строк ICAO 9303 | mrz_block_valid |
| Банковская карта | Regex 12-19 цифр + Луна | luhn_check |
| Расчётный счёт | Regex 20 цифр + контекст или контр. разряд по БИК | account_check |
| БИК | Regex 9 цифр + код страны 04 + контекст | bik_check |
| CVV | Только в окне ключевого слова `cvv/cvc` | — |
| Биометрия | Словарь терминов | — |
| Здоровье/Религия/Политика/Раса | Словари с морфологическими шаблонами | — |

### Контекстный скоринг

Для каждой находки берётся окно ±60 символов и проверяется наличие ключевых слов категории (`паспорт`, `СНИЛС`, `тел`, `email` и т.п.). При совпадении confidence повышается на +0.25 (с верхним порогом 1.0). Это резко снижает ложноположительные срабатывания на «свободных» числах.

### Стратегия для синтетических данных

Многие тестовые данные нарушают контрольные суммы (это нормально для хакатона). Поэтому:

- Если **контрольная сумма валидна** → высокая уверенность.
- Если **контрольная сумма не валидна, но контекст подтверждает** → средняя уверенность (0.7).
- Если **контрольная сумма не валидна и контекста нет** → отбрасываем.

## Извлечение текста

| Формат | Библиотека | Особенности |
|--------|------------|-------------|
| TXT/MD/LOG | chardet + raw read | Авто-определение кодировки |
| HTML | trafilatura → selectolax | Сначала «основной» контент, fallback на чистый текст |
| CSV/TSV | pandas (chunksize=25k) | Стрим, формат `col=val` для контекста детекторам |
| JSON / JSONL | json + рекурсивный обход | Превращение в `key=value` |
| Parquet | pyarrow batch reader | Стрим по 25k записей |
| DOCX | python-docx | Параграфы + ячейки таблиц |
| RTF | striprtf | — |
| DOC | antiword → textutil → libreoffice | Каскадный фолбэк |
| XLS/XLSX | xlrd / openpyxl (read-only) | Лист за листом |
| PDF | pypdf → pdfplumber → OCR | Если pypdf вернул < 200 символов — пробуем pdfplumber, потом OCR |
| Изображения | tesseract (rus+eng) | Greyscale + upscale до 600px по короткой стороне |
| Видео | ffmpeg keyframes → tesseract | Один кадр каждые 2 секунды, до 30 кадров |

## Классификация уровня защищённости

Согласно ПП РФ № 1119:

| Уровень | Условие |
|---------|---------|
| **УЗ-1** | Специальные категории ИЛИ биометрия |
| **УЗ-2** | Платёжные данные ИЛИ много госидентификаторов (≥10 экземпляров или ≥1% записей) |
| **УЗ-3** | Госидентификаторы в небольшом объёме ИЛИ много обычных ПДн (≥50 или ≥10%) |
| **УЗ-4** | Только обычные ПДн в небольшом объёме |
| **Без ПДн** | Ничего не найдено |

Пороги вынесены в `config/default.yaml`.

## Производительность

- **ProcessPoolExecutor** с `workers = CPU` (по умолчанию 8) обрабатывает файлы параллельно.
- **Дедупликация** через xxhash (xxh3-128) — одинаковые файлы (часто встречаются в дампах) сканируются один раз.
- **Streaming для структурированных** — большие CSV/Parquet/JSON читаются батчами без загрузки в память целиком.
- **Лимит чанков на файл** (`max_chunks_per_file=10000`) — защита от взрывных текстов.
- **Ленивая инициализация Natasha** — модели грузятся только при включённом NER.
- **OCR — отдельный путь**, чтобы не блокировать быстрые форматы.

## Безопасность вывода

- Все найденные значения **маскируются** (см. `reporting/masking.py`):
  - `+7 910 ***-**-18` (телефон)
  - `4111 **** **** 1111` (карта)
  - `И. И. И.` (ФИО)
  - и т.п.
- Сохраняется SHA-256 хэш оригинала для трассируемости (без раскрытия значения).
- Конфигурация: `masking: full | partial | hash_only`.
